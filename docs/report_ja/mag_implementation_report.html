概要・背景

mainブランチでは project_type は読み込まれていたものの、gnm（complete ゲノム）の判定にのみ使われていました。MAG（Metagenome-Assembled Genome）や MAG-WGS に特有の DDBJ 提出要件（ENV division、MAG キーワード、isolate qualifier、metagenome_source など）はサポートされていませんでした。

本実装では mainブランチとの差分として以下を新規に追加・変更しました。

＋
MAG / MAG-WGS の project_type 認識と complete フラグ
mag が complete=True を自動設定。project_type を GENOME_CONFIG に格納し、パイプライン全体に伝播。
＋
ENV division と MAG キーワードの自動出力
MAG/MAG-WGS で DIVISION: ENV、KEYWORD: ENV / Metagenome Assembled Genome / MAG を自動追加。
＋
isolate フィールドのサポート
metadata → GENOME_SOURCE_INFORMATION → Genome オブジェクト → source feature → ann ファイルまで isolate を一貫して伝播。
＋
TSV 駆動の条件付きフィールド制御
required_for / excluded_for 列を追加。strain 除外・isolate/metagenome_source/environmental_sample 必須化をコード変更なしで制御。
✎
ff_definition の @@[strain]@@ → @@[isolate]@@ 切り替え
MAG/MAG-WGS では DEFINITION 行の識別子テンプレートを自動切り替え。
✔
boolean フィールドの render() 修正
environmental_sample（boolean 型）が値なしでも valueless qualifier として正しく出力されるよう render ロジックを修正。

MetadataField に追加したメソッド

def is_required_for(self, project_type: str) -> bool:
    return project_type in self.required_for

def is_excluded_for(self, project_type: str) -> bool:
    return project_type in self.excluded_for

def effective_required(self, project_type: str) -> bool:
    if project_type in self.excluded_for: return False
    if project_type in self.required_for:  return True
    return self.mss_required

def should_render(self, project_type: str) -> bool:
    if project_type in self.excluded_for: return False
    if self.value:                          return True
    return self.effective_required(project_type)

boolean フィールドの render() 修正

metadata_util.py

mainブランチでは boolean フィールドの render() は値が "NO" 以外のとき valueless qualifier を出力する実装でした。environmental_sample=TRUE のときは動作しましたが、値が空（未指定）のときは [] を返すため、MAG のプレースホルダー出力ができませんでした。

⚠
render() が [] を返したとき、render_ex_source() 側で should_render() が True なら valueless placeholder 行を追加するフォールバックを実装しています。これにより environmental_sample は値なしでも MAG 提出時に必ず出力されます。
