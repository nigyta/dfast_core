Overview

In the main branch, project_type was parsed from metadata but used only to set complete=True for gnm. DDBJ submission requirements specific to MAG (Metagenome-Assembled Genome) and MAG-WGS — ENV division, MAG keywords, isolate qualifier, metagenome_source, environmental_sample — were not supported.

This implementation adds the following new features relative to the main branch.

＋
MAG / MAG-WGS project_type recognition and complete flag
mag now triggers complete=True automatically. project_type is stored in GENOME_CONFIG and propagated throughout the pipeline.
＋
Automatic ENV division and MAG keywords
For MAG/MAG-WGS, DIVISION: ENV and keywords ENV, Metagenome Assembled Genome, MAG are emitted automatically.
＋
isolate field support
isolate is now propagated end-to-end: metadata file → GENOME_SOURCE_INFORMATION → Genome object → source feature qualifiers → .ann file.
＋
TSV-driven conditional field control
required_for / excluded_for columns added to the TSV. Excluding strain and requiring isolate/metagenome_source/environmental_sample for MAG is now controlled purely via TSV — no code changes needed for future project types.
✎
ff_definition: @@[strain]@@ → @@[isolate]@@ for MAG
The DEFINITION line template in the DDBJ flat file is switched automatically based on project_type.
✔
Boolean field render() fix
environmental_sample (boolean type) now correctly emits a valueless qualifier row even when the value is empty, enabling its use as a MAG placeholder.

New methods on MetadataField

def is_required_for(self, project_type: str) -> bool:
    return project_type in self.required_for

def is_excluded_for(self, project_type: str) -> bool:
    return project_type in self.excluded_for

def effective_required(self, project_type: str) -> bool:
    if project_type in self.excluded_for: return False
    if project_type in self.required_for:  return True
    return self.mss_required  # project_type-independent default

def should_render(self, project_type: str) -> bool:
    if project_type in self.excluded_for: return False  # excluded even if value is set
    if self.value:                          return True   # value present → always render
    return self.effective_required(project_type)         # render placeholder if required

Boolean Field render() Fix

metadata_util.py

In the main branch, the boolean render path emitted a valueless qualifier only when the value was explicitly set (anything other than "NO"). With an empty value it returned [], making it impossible to emit environmental_sample as a placeholder for MAG submissions.

dfc/utils/metadata_util.py — MetadataField.render()
+ if self.type == "boolean": + if str(self.value).lower() in ("yes", "true"): + RET.append(["", "", "", self.qualifier, ""]) + return RET # returns [] when empty; render_ex_source handles placeholder if self.value == "" or self.value == []: RET.append(["", "", "", self.qualifier, ""]) - elif self.type == "boolean" and self.value: - if self.value != "NO": - RET.append(["", "", "", self.qualifier, ""])
⚠
When render() returns [], render_ex_source() adds a fallback valueless placeholder row if should_render() is True. This ensures environmental_sample always appears in MAG output, even when the user has not set a value.
